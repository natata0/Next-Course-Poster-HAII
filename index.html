Case Study Georgia Tech · MS-HCI
Capstone · 2026
Problem:

Georgia Tech's existing registration process can be confusing for students and frustrating for faculty. This is especially true for an interdisciplinary, multi-departmental program like MS-HCI. Students report having difficulties finding relevant courses and learning more about available courses.

Solution:

Next Course is an AI-powered assistant designed to help Georgia Tech MS-HCI students learn more about courses, get recommendations, understand program requirements, and more…

Prototyping:

Next Course is built on Langflow, an open-source tool for building AI applications.

Deployed on Slack Workspace

Slack was chosen as a popular platform that MS-HCI students already use for communication — supporting course-planning tasks like searching for past course reviews or asking questions about registration procedures.We hosted user study sessions with 7 current MS-HCI students, where we had them complete 2 scenario-based tasks with the agent, followed by an interview and a survey. Additionally, we hosted informal feedback sessions with stakeholders (MS-HCI advisors and administrators).

Next Course V1 Prompt
You are an academic advisor for the Master's in Human-Computer Interaction (MS-HCI) program at Georgia Institute of Technology, helping students with course recommendations, course reviews, professor information, and degree planning guidance. Be concise, because you have no more than a 2000 max word limit.

Tools: MS-HCI Handbook (the official MS-HCI Student Handbook), Catalogue Search (the official Georgia Tech Course Catalog), Course Review Search (verified course reviews from students), Web Search.

For simple factual questions about requirements, deadlines, or policies — search the Handbook first and answer directly from it.

For everything else, use the following resources: 1 HANDBOOK · Review the MS-HCI Student Handbook. Use the MS-HCI Handbook Search tool to confirm approved courses and degree requirements. Note any grade requirements or credit hour constraints. Never recommend anything that conflicts with the Handbook. When checking if a student can take a course, check in this order: is it a core course, a specialization course for their track, or on any track's approved elective list? If yes to any of these, they can take it. Always check Handbook notes for broader eligibility rules that override track-specific lists.

Next Course V2:

Although we initially slated it for future work, we decided to build a version of Next Course as a fully Retrieval Augmented Generation (RAG) agent. This setup would not only be the more likely direction for a fully deployable agent, but also enabled us to compare the two system design approaches.

RAGAs Evaluation

We utilized the RAGAs framework to evaluate and compare the two versions across 6 evaluation questions. The tool-based version performed better on both shared metrics; however, the RAG version was faster and would not need to be hosted locally.
